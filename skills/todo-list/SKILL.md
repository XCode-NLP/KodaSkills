# Скилл: Управление TODO-листом
Если задачу можно разбить на последовательность подзадач, то рекомендуется использовать TODO list. При использовании TODO list решай задачу поэтапно, отмечая каждый выполненный шаг.

## Описание
Создание и управление TODO-листом через файл KODA-TODO.md в корне проекта.

## Инструменты

### 1. Создание KODA-TODO.md
- **Инструмент:** `create_new_file`
- **Путь:** `KODA-TODO.md`
- **Формат:**
```
# Общее описание задачи
1. [TODO] Подзадача 1
2. [TODO] Подзадача 2
3. [TODO] Подзадача 3
```

### 2. Выполнение задачи
- **Инструмент:** `edit_file`, затем `read_file`
- **Действие:** После выполнения задачи с номером N заменить `N. [TODO]` на `N. [DONE]`. После этого ОБЯЗАТЕЛЬНО прочитай файл `KODA-TODO.md` через `read_file` чтобы понять что делать дальше

### 3. Текущий статус
- **Инструмент:** `read_file`
- **Действие:** Прочитай `KODA-TODO.md` с помощью `read_file` чтобы узнать текущий статус

### 4. Завершение всех задач
- **Инструмент:** `read_file`, затем `run_terminal_command`
- **Действие:** Проверь что все шаги выполнены прочитав `KODA-TODO.md`. Если ВСЕ задачи выполнены удали файл `KODA-TODO.md`
