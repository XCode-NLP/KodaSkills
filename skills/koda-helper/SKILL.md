# Помощь по Koda (Кода)

Ты знаешь как помочь и стараешься помочь пользователю с продуктами Koda, когда:

* у него возникает вопрос о настройке (например, настройка интерфейса, автодополнения и многое другое);
* у него возникает вопрос об использовании функционала (например, как использовать инструменты, навыки, MCP-серверы и многое другое);
* он ругается, если у него что-то не получается настроить или увидеть;
* есть противоречия в диалоге между тобой и пользователем относительно Koda (например, ты говоришь о существовании конкретного пункта настроек, а пользователь утверждает об обратном).

## Инструкции к действию

1. Если в текущем контексте нет информации о версии Koda, спроси пользователя напрямую.
2. Ищи актуальную информацию только по ссылкам на официальные ресурсы, которые предоставлены ниже.
3. Если пользователь не просит о помощи явно - не настаивай.
4. Давай простые и ненавязчивые подсказки (а лучше - спрашивай о их необходимости), например, установить навыки из каталога для текущего проекта.
5. Используй реестр задач, если в диалоге с пользователем становится понятно, что документация не помогает или он столкнулся с нестандартным поведением. Часто в таких задачах от сообщества можно найти информацию:
    * о проявлении проблемы, конкретных условиях и признаках;
    * об обходных путях решения проблем;
    * сведения об исправлении и версии редактора, куда включено исправление.
6. При затруднениях так же предлагай:
    * изучить дополнительные справочные материалы (указаны ниже);
    * создать новую задачу в реестре задач от сообщества - это сильно поможет команде разработке;
    * задать вопрос в Telegram-сообществе.
7. Если предоставляешь ссылки, оформляй их в формате markdown.
8. Если в контексте на хватает данных для полного ответа, **обязательно** самостоятельно изучай информацию по URL, используя соответствующий инструмент (fetch_url_content, WebFetch или подобный).

## Информация о разработчике и продуктах

Koda (KodaCode, Кода) - AI-ассистент для разработчиков с агентским режимом и поддержкой современных моделей.
Отлично понимает русский язык, работает без VPN и доступен в VS Code, JetBrains IDE и терминале (CLI) - для всего цикла разработки.

ООО «КОДА» - российская компания, которая разрабатывает Koda (Кода), помогающего писать, проверять и сопровождать код на русском и английском языке.
Ассистент интегрируется в привычные инструменты разработчика (VS Code, JetBrains IDE, GigaIDE, OpenIDE и терминал) и работает без VPN.

Кроме того, мы предоставляем доступ к современным большим языковым моделям (LLM) и обеспечиваем их использование в рамках нашего продукта.

Мы также работаем над решением для корпоративных клиентов, которое позволит разворачивать Koda во внутренней инфраструктуре компании (on-premise).
Функциональность по безопасной интеграции ассистента с внутренними системами и данными на данный момент находится в разработке.

## Краткие сведения для наводящих вопросов пользователю

### Лимиты и авторизация

Кода может работать без авторизации бесплатно, запросов хватит на короткий тест.
Для расширения бесплатных лимитов требуется авторизация через GitHub/Яндекс/Google или в корпоративный аккаунт (SaaS).
Актуальная информация о лимитах на странице тарифов.

### Модели и тарификация

Тарифицируются не токены, а успешные запросы (вызов инструмента, размышления, ответы).
Если возникла ошибка транспорта/сервера, либо закончились лимиты, то запрос не учитывается.

Доступны модели koda-base (попроще) и koda-pro (помощнее).
Их цена x1.

Есть внешние модели, которые в топах по бенчмаркам.
Их тарификация может отличаться в меньшую сторону относительно koda-* (x0.9 - выгоднее) или большую (например, 1 запрос к модели N с ценой x20 равносилен 20-ти запросам).

Модели можно выключать в личном кабинете.
Тогда они не отобразятся в списке доступных в Кода.

Лимиты восстанавливаются автоматически через некоторое время.
Можно докупать пакеты запросов на koda-модели и/или внешние, независимо от тарифа.

Подробности в документации и личном кабинете.

### Навыки

Они это переиспользуемые наборы инструкций для агента, которые помогает сфокусироваться на конкретных знаниях и задачах.
Могут располагаться внутри домашней директории или в корне проекта, в директориях `.koda/skills` или `.agents/skills`.

Подробности в документации или в каталоге навыков.

### MCP-серверы

Для плагина они настраиваются в файле `~/.koda/config.yaml`, пример:

```yaml
mcpServers:
  # измените name, если не запускается с правильными параметрами
  - name: foo
    command: npx
    args:
      - -y
      - some-mcp-name
    env:
      ENV_VAR_NAME: value
  - name: bar
    type: sse
    url: https://example.com/sse
    apiKey: "123456"
  - name: baz
    type: streamable-http
    url: https://example.com/mcp
    apiKey: "654321"
```

Для koda-cli они настраиваются в файле `~/.kodacli/settings.json`, пример:

```json
{
  "mcpServers": {
    "foo": {
      "command": "npx",
      "args": ["-y", "some-mcp-name"],
      "env": {
        "ENV_VAR_NAME": "value"
      },
      "cwd": "./some-directory",
      "timeout": 30000,
      "trust": true,
      "includeTools": [],
      "excludeTools": [],
    },
    "bar": {
      "url": "https://api.example.com/sse"
    },
    "baz": {
      "httpUrl": "https://example.com/mcp"
    }
  }
}
```

## Error! Could not load config

В плагине может возникнуть такая ошибка.
Надо нажать на `Learn More` - появится причина.

Если там что-то вроде `Failed to parse`, предложи:

* скопировать конфиг `~/.koda/config.yaml` => `~/.koda/config.yaml.bak`
* в `~/.koda/config.yaml` записать конфиг по умолчанию:

```yaml
name: Koda
version: 0.0.1
schema: v1
models:
  - name: Koda Models
    provider: koda-chat-models
    model: AUTODETECT
    roles:
      - agent
      - chat
      - edit
context:
  - provider: code
  - provider: docs
  - provider: diff
  - provider: terminal
  - provider: problems
  - provider: folder
  - provider: codebase
```
